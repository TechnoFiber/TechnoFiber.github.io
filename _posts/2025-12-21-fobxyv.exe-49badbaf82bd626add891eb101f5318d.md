## Executive Summary

This report documents the analysis of a Windows-based **information-stealing malware**, examined in a fully isolated and controlled lab environment for defensive and educational purposes only.

The malware is designed to harvest **browser credentials, cookies, browsing history, cryptocurrency wallets, messaging application data, system metadata, and screenshots**, followed by **local staging and attempted data exfiltration**.

The sample exhibits **active defense evasion**, including the **automatic termination of analysis and monitoring tools upon execution**, as well as cryptographic protection of collected data.  
No production systems were affected during this analysis.

## Dynamic Analysis

### Execution Behavior

Upon execution, the malware performs the following actions:

- Enumerates running processes
- Actively **terminates monitoring and analysis tools**
- Dynamically loads required Windows libraries
- Spawns multiple threads for parallel collection tasks

Renaming analysis tools (e.g., Procmon) allowed observation, confirming **name-based detection and termination logic**.

---

### File System Activity

The malware creates a dedicated staging directory under:

- C:\ProgramData\{35c7992a-c8fe-11f0-b03e-806e6f6e6963}

Collected data is organized into structured subdirectories, including:

- Browsers
- Wallets
- Messengers
- FTP Clients

Additional files observed:

- Info.txt
- Windows_Info.txt
- Software_Info.txt
- Screenshot.jpg

This structure suggests preparation for operator-side review after exfiltration.

---

### Registry Activity

Observed registry interaction includes:

- HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\fobxyv.exe
- HKCR\AppID\fobxyv.exe
- HKLM\SYSTEM\CurrentControlSet\Services\bam\State\UserSettings\<SID>\fobxyv.exe

These entries create clear forensic artifacts and may support execution tracking or evasion.

---

### Windows Error Reporting (WER) Abuse

The malware invokes WerFault.exe with custom parameters, generating artifacts under:

- C:\ProgramData\Microsoft\Windows\WER\ReportQueue\AppCrash_fobxyv.exe_*

This behavior suggests abuse of Windows Error Reporting to interfere with crash handling or obscure abnormal execution.
